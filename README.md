# ESP32 기반 스마트 주차 관리 시스템

> **Heltec WiFi LoRa 32 V3와 HC-SR04 초음파 센서를 이용하여 주차 공간의 점유 상태를 감지하고 표시하는 마이크로프로세서 팀 프로젝트**

## 프로젝트 개요

이 프로젝트는 실제 주차장에서 활용되는 **스마트파킹 시스템의 핵심 동작 구조**를 소형 모형으로 구현한 프로젝트이다.

각 주차 공간에 설치된 HC-SR04 초음파 센서가 센서와 물체 사이의 거리를 측정하고, ESP32가 측정값을 기준 거리와 비교하여 차량의 존재 여부를 판단한다. 판정 결과는 주차 공간별 적색·녹색 LED로 표시하며, 현재 펌웨어는 보드 내장 OLED, 능동 부저, 시리얼 모니터 출력도 지원한다.

초기 제안에서는 주차 공간 3개를 계획했으나, 최종 제작 과정에서 배선 안정성과 센서 간 간섭 감소, 시연 완성도를 고려하여 **2개 주차 공간을 활성화**하였다.

| 항목 | 내용 |
|---|---|
| 제어 보드 | Heltec WiFi LoRa 32 V3 |
| MCU | ESP32-S3 |
| 감지 대상 | 주차 공간 2개 |
| 감지 센서 | HC-SR04 초음파 센서 2개 |
| 판단 기준 | 평균 측정 거리 40 cm 이하: 차량 있음 |
| 기본 출력 | 적색·녹색 LED |
| 추가 출력 | 보드 내장 OLED, 능동 부저, 시리얼 모니터 |
| 통신 기능 | Wi-Fi·LoRa 미구현 |

---

## 프로젝트 배경

차량 수 증가와 주차 공간 부족으로 인해 운전자가 빈 주차 공간을 찾는 데 많은 시간이 소요될 수 있다. 학교, 아파트, 회사, 쇼핑몰과 같은 공간에서는 주차 구역의 사용 여부를 즉시 확인하기 어려워 불필요한 차량 이동과 내부 혼잡이 발생한다.

본 프로젝트는 이러한 문제를 해결하기 위해 다음 기능을 구현하는 것을 목표로 하였다.

- 주차 공간별 차량 존재 여부 자동 감지
- 운전자가 멀리서도 확인할 수 있는 직관적인 상태 표시
- 저비용 부품을 활용한 소형 스마트 주차 시스템 구현
- 향후 웹·모바일 기반 주차 관리 시스템으로 확장 가능한 구조 설계

---

## 프로젝트의 의의

이 프로젝트는 단순한 거리 측정 실습이 아니라, 실제로 사용되는 스마트파킹 시스템의 기본 처리 흐름을 구현했다는 점에 의미가 있다.

```text
차량 감지
   ↓
거리 데이터 측정
   ↓
기준값과 비교
   ↓
주차 상태 판정
   ↓
LED·OLED·부저를 통한 상태 제공
```

실제 상용 스마트파킹 시스템은 센서, 카메라, 네트워크, 서버 등을 결합하여 대규모 주차장을 관리한다. 본 프로젝트는 그중 **주차면 단위 차량 감지와 상태 표시 기능**을 소형 하드웨어로 축소 구현하였다.

또한 카메라 영상을 수집하지 않고 거리값만 사용하므로 구조가 단순하며, 영상 기반 방식보다 개인정보 침해 가능성이 낮다.

---

## 초음파 센서를 사용한 이유

HC-SR04 초음파 센서를 선택한 주요 이유는 다음과 같다.

1. **거리 측정이 용이함**  
   Trigger 신호를 보낸 뒤 Echo 신호가 돌아오는 시간을 이용해 물체까지의 거리를 직접 계산할 수 있다.

2. **가격이 저렴함**  
   저비용으로 여러 주차 공간에 센서를 배치할 수 있어 교육용 시제품과 소규모 시스템 구현에 적합하다.

3. **비접촉 감지가 가능함**  
   차량이나 물체에 직접 접촉하지 않고 존재 여부를 판단할 수 있다.

4. **ESP32와 연동이 간단함**  
   디지털 GPIO와 `pulseIn()` 함수를 이용해 별도의 복잡한 신호 처리 장치 없이 구현할 수 있다.

5. **카메라 방식보다 구성이 단순함**  
   영상 촬영 장비와 영상 분석 알고리즘이 필요하지 않아 구현 비용과 복잡도를 줄일 수 있다.

다만 센서 설치 각도, 물체 표면, 외부 환경에 따라 측정값이 달라질 수 있으므로 실제 설치 환경에서는 기준 거리 보정이 필요하다.

## 주요 기능

### 주차 공간 감지

- HC-SR04 센서 2개가 각각 하나의 주차 공간을 담당한다.
- 센서별로 거리를 5회 측정한다.
- 정상적으로 측정된 값만 사용해 평균 거리를 계산한다.
- 평균 거리가 40 cm 이하이면 차량이 있는 것으로 판정한다.
- 평균 거리가 40 cm를 초과하면 빈자리로 판정한다.

```text
평균 거리 ≤ 40 cm  → OCCUPIED
평균 거리 > 40 cm  → EMPTY
```

### LED 상태 표시

| 주차 상태 | 적색 LED | 녹색 LED |
|---|---|---|
| 차량 있음 | ON | OFF |
| 빈자리 | OFF | ON |

### OLED 표시

Heltec WiFi LoRa 32 V3 보드에 장착된 **내장 OLED**를 사용한다.

OLED에는 다음 정보가 표시된다.

- 전체 주차 공간 수
- 현재 빈자리 수
- 슬롯별 `FULL` 또는 `EMPTY` 상태
- 슬롯별 측정 거리
- 측정 실패 시 `ERR`

출력 예시:

```text
Smart Parking
Empty: 1 / 2
S1: FULL  18cm
S2: EMPTY 72cm
```

### 부저 알림

| 상황 | 부저 동작 |
|---|---|
| 한 개 이상의 슬롯 상태 변경 | 짧게 1회 |
| 전체 슬롯이 처음 만차 상태가 됨 | 짧게 2회 |
| 만차 상태가 계속 유지됨 | 반복 출력 없음 |
| 만차 해제 후 다시 만차가 됨 | 다시 2회 |

### 시리얼 모니터 출력

각 슬롯의 평균 거리와 판정 결과를 `115200 baud`로 출력한다.

```text
========== Parking Status ==========
Slot 1 : 18.42 cm -> OCCUPIED
Slot 2 : 74.31 cm -> EMPTY
====================================
```

## 시스템 동작 과정

1. ESP32가 초음파 센서, LED, 부저, OLED를 초기화한다.
2. 슬롯 1의 초음파 센서가 거리를 5회 측정한다.
3. 유효한 거리값의 평균을 계산한다.
4. 슬롯 2도 같은 방식으로 순차 측정한다.
5. 평균 거리와 40 cm 기준값을 비교한다.
6. 차량 존재 여부에 따라 적색 또는 녹색 LED를 점등한다.
7. 슬롯 상태가 변경되면 부저를 출력한다.
8. 전체 슬롯이 만차가 되는 순간 만차 알림음을 출력한다.
9. OLED와 시리얼 모니터에 최신 상태를 표시한다.
10. 일정 시간 후 같은 과정을 반복한다.

두 개의 초음파 센서를 동시에 구동하지 않고 순차적으로 측정하여 센서 간 초음파 간섭 가능성을 줄였다.

## 제안 단계와 최종 구현 범위

| 구분 | 초기 제안 | 최종 저장소 기준 |
|---|---|---|
| 주차 공간 | 3개 계획 | 2개 활성화 |
| 차량 감지 | 필수 구현 | 구현 |
| LED 표시 | 필수 구현 | 구현 및 기능시험 완료 |
| OLED 표시 | 추가 구현 | 현재 펌웨어에 포함 |
| 능동 부저 | 선택 구현 | 현재 펌웨어에 포함 |
| 웹페이지 | 확장 구현 | 미구현 |
| Wi-Fi·LoRa 통신 | 확장 가능 | 미사용 |

초기 계획보다 주차 공간 수를 줄인 이유는 다음과 같다.

- 배선 복잡도 감소
- 센서 간 초음파 간섭 가능성 감소
- 회로 안정성 확보
- 제한된 제작 시간 안에서 시연 완성도 향상

> 최종 결과보고서의 기능시험은 초음파 센서와 LED를 중심으로 수행되었다. 현재 저장소의 펌웨어에는 이후 정리된 OLED와 부저 제어 코드가 포함되어 있다.

## 핀 연결

### 전체 핀맵

| 기능 | 슬롯 1 | 슬롯 2 |
|---|---:|---:|
| HC-SR04 TRIG | GPIO26 | GPIO33 |
| HC-SR04 ECHO | GPIO35 | GPIO36 |
| 적색 LED | GPIO1 | GPIO2 |
| 녹색 LED | GPIO4 | GPIO5 |
| Active Buzzer | GPIO7 | - |
| OLED SDA | GPIO17 | - |
| OLED SCL | GPIO18 | - |

### HC-SR04 연결

| HC-SR04 핀 | 연결 |
|---|---|
| VCC | 5V |
| GND | GND |
| TRIG | 슬롯별 TRIG GPIO |
| ECHO | 전압 분배 회로를 거쳐 슬롯별 ECHO GPIO |

HC-SR04의 Echo 신호는 일반적으로 5V 수준이므로 ESP32 GPIO에 직접 연결하지 않는다. 다음과 같이 전압 분배 회로를 구성하여 입력 전압을 낮춘다.

```text
HC-SR04 ECHO
     |
    1 kΩ
     |
     +------ ESP32 ECHO GPIO
     |
    2 kΩ
     |
    GND
```

### LED 연결

```text
ESP32 GPIO → 220 Ω 저항 → LED 애노드(+)
LED 캐소드(-) → GND
```

### Active Buzzer 연결

```text
Buzzer Signal → GPIO7
Buzzer GND    → GND
```

사용하는 부저의 소비 전류가 ESP32 GPIO 허용 범위를 초과하면 트랜지스터 구동 회로를 추가해야 한다.

## 주요 설정값

```cpp
const int SLOT_COUNT = 2;
const float THRESHOLD_CM = 40.0;
const unsigned long ECHO_TIMEOUT_US = 30000;
const int SAMPLE_COUNT = 5;
```

| 설정값 | 의미 |
|---|---|
| `SLOT_COUNT` | 활성화된 주차 공간 수 |
| `THRESHOLD_CM` | 차량 존재 여부를 판단하는 기준 거리 |
| `ECHO_TIMEOUT_US` | Echo 신호 최대 대기 시간 |
| `SAMPLE_COUNT` | 평균 계산에 사용하는 측정 횟수 |

OLED 또는 부저 기능을 사용하지 않으려면 다음 값을 변경한다.

```cpp
#define USE_OLED 0
#define USE_BUZZER 0
```

## 실행 방법

1. Arduino IDE에 ESP32 보드 패키지를 설치한다.
2. `Adafruit GFX Library`와 `Adafruit SSD1306`을 설치한다.
3. 회로를 핀맵에 맞게 연결한다.
4. Arduino IDE에서 `Parking.ino` 파일을 연다.
5. 보드를 `Heltec WiFi LoRa 32(V3)`로 선택한다.
6. 연결된 포트를 선택한다.
7. 코드를 컴파일하고 보드에 업로드한다.
8. 시리얼 모니터를 `115200 baud`로 연다.
9. 센서 앞에 물체를 놓거나 제거하여 LED와 OLED 상태를 확인한다.

실제 모형의 크기와 센서 설치 위치가 다르면 `THRESHOLD_CM` 값을 다시 측정하여 조정해야 한다.

## 기능시험 결과

두 개의 센서 앞에 물체를 놓거나 제거하여 가능한 네 가지 상태 조합을 시험하였다.

| 시험 | 주차 공간 1 | 주차 공간 2 | 기대 LED | 관찰 결과 |
|---:|---|---|---|---|
| 1 | 빈자리 | 빈자리 | 초록 / 초록 | 정상 |
| 2 | 차량 있음 | 빈자리 | 빨강 / 초록 | 정상 |
| 3 | 빈자리 | 차량 있음 | 초록 / 빨강 | 정상 |
| 4 | 차량 있음 | 차량 있음 | 빨강 / 빨강 | 정상 |

모형 환경의 기본 기능시험에서는 **4개 상태 조합 중 4개가 기대한 LED 출력과 일치**하였다.

이 결과는 차량 있음과 빈자리의 기본 상태 조합을 구분할 수 있음을 보여준다. 다만 실제 차량 환경의 장시간 정확도, 오검출률, 기상 조건에 대한 성능을 의미하지는 않는다.

## 현실적 제한사항

- 실제 주차장이 아닌 교육용 모형 환경에서 검증하였다.
- 초음파 센서의 설치 각도와 물체 표면에 따라 측정값이 달라질 수 있다.
- 기준 거리 주변에서는 상태가 반복적으로 변경될 수 있다.
- 두 센서를 순차 측정하지만 설치 간격에 따라 초음파 간섭이 발생할 수 있다.
- 브레드보드와 점퍼선 구조이므로 충격과 장시간 사용에 취약하다.
- 방수·방진 구조와 보호 케이스가 적용되지 않았다.
- Wi-Fi, LoRa, 데이터베이스, 웹·모바일 모니터링 기능은 구현하지 않았다.
- 실제 차량을 이용한 장시간 반복시험과 통계적 정확도 평가는 수행하지 않았다.

## 개선 방향

- 차량 진입과 이탈 임계값을 다르게 설정하는 히스테리시스 적용
- 중앙값 필터 또는 이동평균 필터 적용
- 연속된 동일 판정이 일정 횟수 이상일 때만 상태 변경
- 주차 환경별 자동 기준 거리 보정
- 방수형 거리 센서와 보호 케이스 적용
- 브레드보드 회로를 PCB 또는 납땜 기판으로 변경
