# ESP32 기반 스마트 주차 관리 시스템

## 프로젝트 한 줄 요약

Heltec WiFi LoRa 32 V3와 HC-SR04 초음파 센서를 사용해 2개 주차 공간의 점유 상태를 감지하고 LED, OLED, 부저, 시리얼 모니터로 표시하는 Arduino 기반 시제품이다.

## 프로젝트 목적

이 프로젝트는 스마트 주차 시스템의 기본 동작 흐름을 소형 하드웨어로 구현하는 것을 목표로 한다. 초음파 센서로 차량과 센서 사이의 거리를 측정하고, 기준 거리와 비교해 각 주차 공간이 비어 있는지 또는 점유되었는지 판단한다.

현재 코드는 발표 및 시연용 시제품 기준으로 작성되었으며, 실제 주차장 운영 환경에 바로 적용하기 위한 완성형 시스템은 아니다.

## 주요 기능

- HC-SR04 초음파 센서 2개를 이용한 주차 슬롯별 거리 측정
- 평균 거리값 기반 주차 여부 판정
- 적색·녹색 LED를 이용한 슬롯 상태 표시
- 보드 내장 OLED를 이용한 빈자리 수와 슬롯별 상태 표시
- 능동 부저를 이용한 상태 변경 및 만차 알림
- 시리얼 모니터를 통한 측정값과 판정 결과 출력

## 사용 기술

| 항목 | 내용 |
|---|---|
| 보드 | Heltec WiFi LoRa 32 V3 |
| MCU | ESP32-S3 |
| 개발 환경 | Arduino IDE |
| 언어 | Arduino C/C++ |
| 센서 | HC-SR04 초음파 센서 2개 |
| 출력 장치 | 적색 LED 2개, 녹색 LED 2개, OLED, 능동 부저 |
| 통신 | 현재 코드 기준 Wi-Fi·LoRa 미사용 |

## 현재 코드 기준 주요 규격

| 항목 | 값 |
|---|---|
| 주차 슬롯 수 | 2 |
| 차량 감지 기준 | 평균 거리 40 cm 이하 |
| 센서별 반복 측정 횟수 | 5회 |
| Echo 대기 제한 시간 | 30,000 us |
| 시리얼 통신 속도 | 115200 baud |

세부 핀 배치, 코드 흐름, 라이브러리 정보는 [Parking/README.md](Parking/README.md)를 참고한다.

## 실행 또는 빌드 방법

1. Arduino IDE에서 ESP32 보드 패키지를 설치한다.
2. 라이브러리 매니저에서 다음 라이브러리를 설치한다.
   - `Adafruit GFX Library`
   - `Adafruit SSD1306`
   - `Adafruit BusIO`
3. Arduino IDE에서 `Parking/Parking.ino`를 연다.
4. 보드를 `Heltec WiFi LoRa 32(V3)`로 선택한다.
5. 실제 연결된 포트를 선택한다.
6. 컴파일 후 보드에 업로드한다.
7. 시리얼 모니터를 `115200 baud`로 열어 측정값과 판정 결과를 확인한다.

현재 저장소에는 Arduino CLI 설정 파일이 포함되어 있지 않다. 빌드는 Arduino IDE에서 수행하는 방식을 기준으로 문서화한다.

## 테스트 및 검증 방법

현재 검증은 시제품 하드웨어에서 각 센서 앞에 물체를 놓거나 제거하며 LED 상태가 예상과 일치하는지 확인하는 방식으로 수행한다.

확인해야 할 기본 조합은 다음과 같다.

| 슬롯 1 | 슬롯 2 | 기대 동작 |
|---|---|---|
| 비어 있음 | 비어 있음 | 두 슬롯 모두 녹색 LED |
| 점유 | 비어 있음 | 슬롯 1 적색 LED, 슬롯 2 녹색 LED |
| 비어 있음 | 점유 | 슬롯 1 녹색 LED, 슬롯 2 적색 LED |
| 점유 | 점유 | 두 슬롯 모두 적색 LED, 만차 알림 |

빌드 성공은 실제 하드웨어 동작을 보장하지 않는다. 발표 또는 시연 전에는 센서 거리값, LED 출력, OLED 표시, 부저 동작을 실제 장치에서 다시 확인해야 한다.

## 주의사항

- HC-SR04 Echo 신호는 일반적으로 5V이므로 ESP32 입력 핀에는 전압 분배 회로를 거쳐 연결해야 한다.
- `THRESHOLD_CM` 값은 시제품 기준값이다. 실제 설치 높이, 센서 각도, 차량 형태에 따라 재조정이 필요하다.
- 현재 코드는 Wi-Fi, LoRa, 데이터베이스, 웹·모바일 연동을 구현하지 않는다.
- 브레드보드 기반 시제품은 진동, 배선 접촉, 전원 안정성에 취약할 수 있다.
