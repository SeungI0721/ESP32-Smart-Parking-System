# Parking 폴더

## 이 폴더의 역할

이 폴더는 ESP32 기반 스마트 주차 시제품의 Arduino 스케치를 관리한다. 주차 슬롯 2개의 초음파 센서 값을 측정하고, 측정 결과를 LED, OLED, 부저, 시리얼 모니터로 출력한다.

## 주요 파일

| 파일 | 설명 |
|---|---|
| `Parking.ino` | 센서 측정, 주차 상태 판정, LED/OLED/부저 출력 제어를 담당하는 Arduino 스케치 |

## 실행 방법

1. Arduino IDE에서 `Parking.ino`를 연다.
2. 보드를 `Heltec WiFi LoRa 32(V3)`로 선택한다.
3. `Adafruit GFX Library`, `Adafruit SSD1306`, `Adafruit BusIO`가 설치되어 있는지 확인한다.
4. 하드웨어를 핀 배치에 맞게 연결한다.
5. 컴파일 후 보드에 업로드한다.
6. 시리얼 모니터를 `115200 baud`로 열어 측정값과 상태 판정을 확인한다.

## 코드 흐름

```text
setup()
├─ 시리얼 통신 초기화
├─ 초음파 센서, LED, 부저 핀 초기화
└─ OLED 초기화

loop()
├─ 슬롯 1 거리 측정
├─ 슬롯 1 점유 상태 판정
├─ 슬롯 2 거리 측정
├─ 슬롯 2 점유 상태 판정
├─ LED 상태 갱신
├─ 상태 변경 또는 만차 여부에 따라 부저 출력
├─ OLED 화면 갱신
└─ 시리얼 모니터 출력
```

초음파 센서는 순서대로 측정한다. 두 센서를 동시에 구동하면 반사파 간섭이 생길 수 있으므로, 각 측정 사이에 짧은 대기 시간을 둔다.

## 외부 의존성

| 의존성 | 용도 |
|---|---|
| `Arduino.h` | Arduino 기본 함수와 타입 |
| `Wire` | OLED I2C 통신 |
| `Adafruit GFX Library` | OLED 그래픽 출력 기반 |
| `Adafruit SSD1306` | SSD1306 OLED 제어 |
| `Adafruit BusIO` | Adafruit 라이브러리 의존성 |

`USE_OLED` 값을 `0`으로 변경하면 OLED 관련 코드를 컴파일 대상에서 제외할 수 있다.

## 주요 설정값

| 항목 | 값 | 설명 |
|---|---:|---|
| `SLOT_COUNT` | 2 | 감지할 주차 슬롯 수 |
| `THRESHOLD_CM` | 40.0 | 이 거리 이하이면 차량이 있는 것으로 판정 |
| `ECHO_TIMEOUT_US` | 30000 | Echo 신호 최대 대기 시간 |
| `SAMPLE_COUNT` | 5 | 센서별 반복 측정 횟수 |
| `USE_OLED` | 1 | OLED 출력 사용 여부 |
| `USE_BUZZER` | 1 | 부저 출력 사용 여부 |

`THRESHOLD_CM`은 현재 시제품 기준값이다. 실제 센서 위치나 차량 높이가 바뀌면 다시 측정해 조정해야 한다.

## 핀 배치

| 기능 | 슬롯 1 | 슬롯 2 |
|---|---:|---:|
| HC-SR04 TRIG | GPIO26 | GPIO33 |
| HC-SR04 ECHO | GPIO35 | GPIO36 |
| 적색 LED | GPIO1 | GPIO2 |
| 녹색 LED | GPIO4 | GPIO5 |

| 기능 | 핀 |
|---|---:|
| Active Buzzer | GPIO7 |
| OLED SDA | GPIO17 |
| OLED SCL | GPIO18 |

HC-SR04 Echo 핀은 5V 신호가 출력될 수 있다. ESP32 GPIO 보호를 위해 전압 분배 회로를 사용한다.

## 테스트 방법

1. 업로드 후 시리얼 모니터를 `115200 baud`로 연다.
2. 각 센서 앞에 물체를 놓거나 제거한다.
3. 시리얼 모니터의 거리값과 `OCCUPIED` 또는 `EMPTY` 판정을 확인한다.
4. LED 색상이 판정 결과와 일치하는지 확인한다.
5. 두 슬롯이 모두 점유되었을 때 만차 부저가 한 번 발생하는지 확인한다.
6. OLED에 빈자리 수와 슬롯별 상태가 표시되는지 확인한다.

## 주의사항

- 측정 실패 시 해당 슬롯의 기존 점유 상태를 유지한다. 순간 오류로 상태가 잘못 바뀌는 것을 줄이기 위한 처리이다.
- 초음파 센서는 설치 각도, 물체 표면, 주변 환경에 따라 값이 흔들릴 수 있다.
- 브레드보드와 점퍼선 연결 상태가 불안정하면 LED, OLED, 센서 값이 비정상적으로 보일 수 있다.
- 현재 코드는 Wi-Fi와 LoRa 전송 기능을 사용하지 않는다.
